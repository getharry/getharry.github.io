# 什么是CSS中不好的注释？
       
因为css的地位的原因，总是让大多数程序员忽视它的注释的作用，但当你遇见复杂的sass代码，如果命名还不规范的话，我想你会痛不欲生。好了，我们先来看一些官方版本的注释。    
出自于bootstrap3的早期版本：

```css
// Addresses
address {…}

// Unordered and Ordered lists
ul,
ol {…}

// Blockquotes
blockquote {…}
```
赶紧停！
千万不要写那种注释，赶紧删掉这些多余的东西，它仅仅是在重复代码而已。哈哈，当然，新版本的Bootstrap已经删除掉大部分多此一举的无用注释了。
